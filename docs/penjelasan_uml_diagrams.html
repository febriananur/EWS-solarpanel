1. Pendahuluan — Gambaran Umum Sistem

Early Warning System (EWS) untuk Panel Surya adalah sistem peringatan dini berbasis Machine Learning (Linear Regression) yang dirancang untuk mendeteksi potensi kebakaran pada panel surya akibat overheating. Sistem ini menggunakan sensor suhu dan kelembaban DHT22 yang terpasang pada modul ESP32, dan memanfaatkan pendinginan Peltier sebagai mekanisme mitigasi otomatis.

📌 Tujuan Utama Sistem
  1. Mengumpulkan data suhu & kelembaban secara real-time dari sensor DHT22 via ESP32
  2. Memprediksi tren suhu ke depan menggunakan algoritma Linear Regression
  3. Mengklasifikasikan level bahaya (Normal → Warning → Danger → Critical)
  4. Mengirimkan notifikasi peringatan melalui Telegram Bot dan Web Dashboard
  5. Mengontrol sistem pendinginan Peltier secara otomatis maupun manual

Komponen Utama Sistem

🔧 Hardware

ESP32 microcontroller dengan sensor DHT22 (suhu + kelembaban) dan modul Peltier Cooler. ESP32 membaca sensor dan mengirim data ke Firebase.

prototype_server.ino

🗄️ Database

Firebase Realtime Database sebagai jembatan data antara ESP32 (hardware) dan Node.js server (backend). Menyimpan data sensor, riwayat, alert, dan kontrol.

⚙️ Backend

Node.js server dengan Express + Socket.IO. Menjalankan analisis Linear Regression, klasifikasi bahaya, dan mengelola notifikasi secara periodik.

src/server.js

📊 Frontend

Web Dashboard real-time yang menampilkan data sensor, grafik tren suhu, prediksi, level bahaya, dan kontrol Peltier. Update via Socket.IO.

public/index.html

Diagram sekuens ini memvisualisasikan alur eksekusi fungsi utama processEarlyWarning() yang dijalankan secara periodik oleh cron scheduler. Fungsi ini merupakan jantung dari sistem EWS — mengkoordinasikan seluruh pipeline dari pengambilan data hingga penyiaran notifikasi.

3.2 Langkah-langkah Eksekusi (Step-by-Step)

StepDari → KeAksiPenjelasan
1 Cron → Server processEarlyWarning() Scheduler memicu fungsi utama. Di mode produksi, ini berjalan setiap 1 menit via node-cron. Di mode demo, setiap 5 detik via setInterval.
2 Server → Firebase getHistoricalData(deviceId, 30) Mengambil 30 data point terakhir dari /devices/{id}/history. Data diurutkan berdasarkan timestamp dan dinormalisasi dari millis() ESP32 ke Unix timestamp.
3 Firebase → Server Return history[] Array berisi 30 objek data sensor (suhu, kelembaban, timestamp) yang sudah dinormalisasi.
4 Server → Firebase getCurrentReading(deviceId) Mengambil data sensor terkini dari /devices/{id}/latest.
5 Firebase → Server Return currentReading Objek data sensor terakhir (suhu aktual, kelembaban, status Peltier, dll).
6 Server → LR train(history) Melatih model Linear Regression dengan data historis. Menghitung slope (laju kenaikan suhu per menit), intercept, dan R² (akurasi model).
7 LR → Server Return modelStats Objek berisi: slope, intercept, rSquared, interpretation, dan calculationLogs.
8 Server → LR predict(PREDICTION_MINUTES) Memprediksi suhu N menit ke depan (default: 10 menit) berdasarkan model yang sudah dilatih. Menggunakan formula: y = mx + b.
9 LR → Server Return prediction Objek berisi: predictedTemperature, confidence (R²×100), trend (naik/turun/stabil), ratePerMinute.
10 Server → LR minutesToThreshold(50, currentTemp) Menghitung estimasi berapa menit lagi suhu mencapai threshold kritis (50°C). Formula: t = (threshold - current) / slope.
11 LR → Server Return minutesToDanger Angka menit (atau null jika suhu sedang turun / stabil).
12 Server → Alert classifyDanger(current, prediction, modelStats) Mengklasifikasikan level bahaya berdasarkan 3 faktor: suhu aktual, prediksi suhu, dan kelembaban. Menghasilkan level tertinggi dari ketiga evaluasi.
13 Alert → Server Return dangerInfo Objek berisi: level (normal/warning/danger/critical), tempLevel, humidityRisk, predictionRisk, dan array factors.
14 Server → Alert generateAlert(dangerInfo, ...) Membuat objek alert terformat dengan judul, pesan, detail, dan tindakan yang diperlukan. Return null jika level = normal.
15 Alert → Server Return alertObject Objek alert lengkap (atau null). Berisi: level, title, message, action, sensorData, prediction, timestamp.
⤷ Kondisional: Jika alertObject !== null (terdeteksi bahaya):
16a Server → Firebase saveAlert(deviceId, alertObject) Menyimpan alert ke Firebase di path /devices/{id}/alerts untuk logging.
16b Server → Telegram notify(alertObject, ...) Mengirim notifikasi ke Telegram. Dilengkapi rate limiting agar tidak spam (warning: 30 menit, danger: 15 menit, critical: 5 menit).
⤷ Jika alertObject === null (status normal): Memanggil telegram.onStatusNormal() untuk mereset recovery cooldown.
17 Server → Dashboard io.emit('sensor-update', latestState) Menyiarkan seluruh state terkini (sensor, model, prediksi, bahaya, chart) ke semua dashboard yang terhubung via Socket.IO.
📌 Catatan Penting

Seluruh proses dari Step 1 hingga Step 17 berjalan dalam satu siklus eksekusi processEarlyWarning(). Di mode produksi, siklus ini berulang setiap 1 menit. Selain trigger periodik, siklus juga dipicu secara real-time oleh Firebase listener (subscribeToSensor) setiap kali ESP32 mengirim data baru.

Properties (Atribut)

AksesNamaTipePenjelasan

PublicinterceptNumberTitik potong sumbu Y (b). Representasi suhu awal saat t=0.

Methods (Fungsi)

AksesMethodReturnPenjelasan
Publictrain(dataPoints)ObjectMelatih model dengan data historis sensor. Menghitung slope & intercept menggunakan formula Least Squares: m = (nΣxy - ΣxΣy) / (nΣx² - (Σx)²). Minimal memerlukan 3 data point.
Publicpredict(minutesAhead)ObjectMemprediksi suhu N menit ke depan menggunakan formula: y = mx + b. Mengembalikan predictedTemperature, confidence, trend, dan ratePerMinute.
PublicminutesToThreshold(thresholdTemp, currentTemp)Number|nullMenghitung estimasi waktu (menit) hingga suhu mencapai threshold tertentu. Formula: t = (threshold - current) / slope. Return null jika slope ≤ 0 (suhu tidak naik).
PublicgetRegressionLine(numPoints)ArrayMenghasilkan titik-titik untuk menggambar garis regresi pada chart, termasuk 5 titik prediksi ke depan.
Private_calculateRSquared(points)ObjectMenghitung R² (Coefficient of Determination). R² = 1 - (SSres / SStot). Digunakan internal oleh train().
Private_interpretSlope()StringMenginterpretasikan slope menjadi teks deskriptif (contoh: "Suhu naik cepat (~12.0°C/jam)").

5. Diagram 4: Flowchart — Logika Klasifikasi Bahaya

Flowchart ini menggambarkan secara spesifik bagaimana fungsi classifyDanger() di alertSystem.js menentukan level bahaya akhir berdasarkan penggabungan tiga faktor evaluasi: suhu saat ini, prediksi suhu, dan kelembaban.

5.1 Tahap 1: Evaluasi Suhu Saat Ini

Langkah pertama adalah mengevaluasi suhu aktual yang dibaca dari sensor DHT22:

KondisiLevelKeterangan
Suhu ≥ 50°CCRITICALSuhu ekstrem. Risiko kebakaran sangat tinggi. Komponen bisa rusak permanen.
Suhu ≥ 40°CDANGEROverheat. Panel surya beroperasi di luar batas aman. Penurunan efisiensi drastis.
Suhu ≥ 30°CWARNINGMulai panas. Suhu melewati batas nyaman. Perlu pemantauan lebih ketat.
Suhu < 30°CNORMALSuhu aman. Panel beroperasi dalam batas normal (suhu ruang / pendinginan aktif).

5.2 Tahap 2: Evaluasi Prediksi Suhu

Langkah kedua menggunakan hasil prediksi dari model Linear Regression. Ini memberikan peringatan proaktif — sebelum suhu benar-benar mencapai level berbahaya:

KondisiRisk LevelKeterangan
Prediksi ≥ 50°CCRITICALModel memprediksi suhu akan mencapai zona kritis dalam waktu dekat.
Prediksi ≥ 40°CDANGERModel memprediksi suhu akan memasuki zona bahaya.
Prediksi ≥ 30°CWARNINGModel memprediksi suhu akan meningkat ke zona waspada.
Prediksi < 30°CNONETidak ada risiko dari prediksi. Suhu diprediksi tetap aman.
⚠️ Mengapa Evaluasi Prediksi Penting?

Tanpa prediksi, sistem hanya bisa bereaksi setelah suhu sudah berbahaya. Dengan Linear Regression, sistem bisa memberikan peringatan sebelum suhu mencapai batas kritis — memberikan waktu bagi operator untuk mengambil tindakan pencegahan.

5.3 Tahap 3: Evaluasi Kelembaban

Kelembaban rendah meningkatkan risiko kebakaran karena kondisi kering mempercepat penyebaran api:

KondisiRisk LevelKeterangan
Kelembaban < 10%CRITICALSangat kering. Risiko kebakaran meningkat drastis. Bahan mudah terbakar.
Kelembaban < 20%WARNINGKering. Kondisi perlu diwaspadai, terutama jika dikombinasikan dengan suhu tinggi.
Kelembaban ≥ 20%NORMALKelembaban cukup. Tidak ada risiko tambahan dari faktor kelembaban.
